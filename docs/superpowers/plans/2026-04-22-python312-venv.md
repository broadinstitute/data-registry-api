# Python 3.12 + Venv Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Migrate `data-registry-api` from Python 3.9 (bare pip) to Python 3.12 virtual environments on the deployment server, and update CI to match.

**Architecture:** Add a `get_venv_dir` helper to the fabfile, update `setup` to create a Python 3.12 venv and pip install before installing the service file, update `migrate` to use venv binaries, update both service files to point `ExecStart` at the venv python, and bump CI to Python 3.12. `/usr/bin/python3.12` is already installed on the server (Amazon Linux 2023, via `sudo dnf install python3.12`). Cutover is `fab setup` per environment.

**Tech Stack:** Python/Fabric 2, systemd, Amazon Linux 2023 EC2, GitHub Actions

**Spec:** `docs/superpowers/specs/2026-04-22-python312-venv-design.md`

- [ ] **Step 1: Add `get_venv_dir` helper**

In `deploy/fabfile.py`, add this function immediately after `get_checkout_directory` (currently at line 75):

```python
def get_venv_dir(env):
    return get_checkout_directory(env) + "/venv"
```

- [ ] **Step 2: Update the `setup` task**

Replace the entire `setup` function with:

```python
@task
def setup(c, env):
    """
    Install and enable the systemd service for the specified environment.
    Run once to set up the service on the server, or again to update the
    service file. Creates the Python 3.12 venv and installs dependencies.

    Parameters:
        c: The connection context.
        env: The environment (e.g., 'dev', 'prd').
    """
    checkout_dir = get_checkout_directory(env)
    venv_dir = get_venv_dir(env)
    service = "dr-api-dev" if env == 'dev' else "dr-api-prd"

    # Create venv and install dependencies
    c.run(f"/usr/bin/python3.12 -m venv {venv_dir}")
    with c.cd(checkout_dir):
        c.run(f"{venv_dir}/bin/pip install -r requirements.txt")

    # Install service file
    deploy_dir = os.path.dirname(os.path.abspath(__file__))
    c.put(os.path.join(deploy_dir, f"{service}.service"), f"/tmp/{service}.service")
    c.run(f"sudo mv /tmp/{service}.service /etc/systemd/system/{service}.service")
    c.run("sudo systemctl daemon-reload")
    c.run(f"sudo systemctl enable {service}")
    c.run(f"sudo systemctl restart {service}")
```

- [ ] **Step 3: Update the `migrate` task**

Replace the entire `migrate` function with:

```python
@task
def migrate(c, env):
    """
    Run db migrations.
    """
    directory = get_checkout_directory(env)
    venv_dir = get_venv_dir(env)
    with c.cd(directory):
        db = "dataregistry_qa" if env == 'dev' else "dataregistry"
        c.run(f"{venv_dir}/bin/pip install -r requirements.txt")
        c.run(f"export DATA_REGISTRY_DB_NAME={db}; {venv_dir}/bin/python -m alembic upgrade head")
```
